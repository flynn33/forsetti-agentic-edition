# Accountability Policy

**Governing Authority**: `COMPLIANCE_POLICY.md`
**Policy Level**: Compliance support policy
**Approval Class**: Governance-Class

## Purpose

This policy preserves accountable human governance evidence while prohibiting tool attribution. Accountability belongs to the accountable human owner and governed role operating under a task contract or remediation phase.

## Prohibited Attribution

Repository artifacts must not credit tools, models, vendors, automation, or agents as authors, contributors, reviewers, validators, approvers, releasers, maintainers, or sources of work.

The prohibition applies to source files, generated code comments, commit messages, changelog entries, release notes, README notices, contributor lists, authorship metadata, documentation prose, generated artifacts, and review text intended for publication.

## Required Accountability Evidence

Governed work must record:

- accountable human owner;
- acting governed role;
- contract ID or remediation phase ID;
- review evidence;
- validation evidence;
- approval evidence when required.

Accountability evidence is not attribution credit. It identifies who is responsible under the governance model and which evidence supports the decision.

## Rejection Conditions

Work must be rejected or marked for changes when it adds attribution credit, omits accountable owner or governed-role evidence, treats a tool as an authority, or substitutes tool output for required review and validation evidence.
